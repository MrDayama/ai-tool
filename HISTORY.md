# 開発経緯と主な変更点 (Development History & Key Changes)

このプロジェクトは、AIエージェントを活用した統合ダッシュボードの構築を目的としています。
当初のポートフォリオ/ビジネス向けの機能（カレンダー、メモ、To-Do、GitHub連携）から、より高度なタスク管理とAI自律開発機能へと進化してきました。

## 🛠️ 主なアップデート履歴

### 1. セキュリティ強化と履歴のクリーンアップ
- **APIキーの秘匿化**: Gemini API等の機密情報をコードから削除し、環境変数（`.env`）での管理に移行。
- **Git履歴の抹消**: 過去のコミットに含まれていた機密情報を完全に削除。
- **バイナリファイルの除外**: GitHubの容量制限（100MB）を超えるビルド成果物（`ai-agent.exe`, `build/`）をGit管理から除外し、`.gitignore` を最適化。

### 2. タスク管理システムの高度化（チケット制）
- **階層構造（親子チケット）**: 親タスクと子タスクの紐付け機能を実装。
- **進捗率（0-100%）とステータス**: すべてのタスクに進捗率を導入。10%刻みで直感的に設定可能。
- **担当者割り当て**: 複数人が利用することを想定し、担当者ごとのフィルタリング機能を実装。

### 3. UI/UXの刷新と独立画面化
- **タブナビゲーション**: 画面上部の切り替えにより、各機能（カレンダー、チケット管理、ガントチャート）を広々と使えるように変更。
- **ガントチャートの強化**: チケットの期間と進捗率をリアルタイムに反映し、バー内部での進捗視認化を実装。
- **カレンダーの最適化**: 予定管理に集中できるよう、カレンダー画面からチケット情報の表示を削除。
- **完全レスポンシブデザイン**: すべての画面で横スクロールを解消し、画面幅に収まるようレイアウトを再設計。スマホ向けにチケットのカード表示を導入。

### 4. GitHub Actions (CI/CD) の導入
- **AI Autonomous Developer**: 特定のファイル（`.ai/instructions.md`）の変更を検知してAIエージェントを自動実行し、コードを自動修正・機能追加するワークフローを構築。

## 📄 ファイル構成と役割
- `index.html`: ダッシュボードのメイン構造（タブ・各画面・モーダル）。
- `app.js`: カレンダー、チケット管理、ガントチャート、ナビゲーションのロジック。
- `style.css`: サイバーパンクなデザインシステムとレスポンシブなレイアウト。
- `ai.js`: Gemini APIを通じたAI要件定義・継続生成ロジック。
- `.github/workflows/ai-worker.yml`: CI/CD ワークフロー定義。
- `ai-agent/`: Pythonベースの自律型開発エージェントのソース。
